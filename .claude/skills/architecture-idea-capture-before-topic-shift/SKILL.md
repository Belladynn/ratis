# architecture-idea-capture-before-topic-shift

A useful architecture idea surfaces mid-conversation, but the user signals
it's not the current subject. Expanding it derails the active objective;
dropping it loses it. This skill is the discipline to park the idea in a
durable location in one short entry, then return immediately. The holding
location is whatever backlog the project uses — `DECISIONS_PENDING.md`, a
kanban, an issue tracker — not hardcoded to one file.

## When to Use

- A useful architecture tangent emerges but the user says it's not the
  current subject.
- You're about to shift topics and there's a valuable side idea worth
  preserving.

## When NOT to Use

- The idea *is* the current objective — pursue it, don't park it.
- A trivial or already-tracked thought — don't clutter the backlog with
  noise or duplicates.
- The user wants to discuss it now — capturing-and-dropping would ignore
  an explicit request.

## Procedure

1. **Pick the holding location.** Use the project's decision backlog /
   kanban / tracker; ask or infer it if ambiguous, rather than defaulting
   blindly to one file.
2. **Write one durable entry.** Capture a short name, the rationale (why
   it's worth keeping), and a concrete next action — enough that it's
   actionable later without this conversation's context.
3. **Return immediately.** Go straight back to the main topic without
   expanding the tangent further.
